# 🌾 AI Drought Predictor - Система прогнозирования засухи

[![Python](https://img.shields.io/badge/Python-3.12-blue.svg)](https://www.python.org/)
[![TensorFlow](https://img.shields.io/badge/TensorFlow-2.16-orange.svg)](https://www.tensorflow.org/)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

**AI Drought Predictor** — интеллектуальная система прогнозирования засушливого года на основе рекуррентных нейронных сетей (LSTM) и методов глубокого обучения. Предназначена для анализа климатических данных и своевременного предупреждения о рисках засухи в сельскохозяйственных регионах.

## 📋 Описание проекта

Изменение климата приводит к учащению экстремальных метеорологических явлений, особенно в южных регионах. Продолжительные засухи наносят существенный ущерб сельскому хозяйству. Система AI Drought Predictor решает задачу своевременного и надёжного прогнозирования засушливого года, что позволяет заранее планировать адаптационные меры: корректировать посевные площади, оптимизировать водопользование и принимать управленческие решения.

Традиционные методы оценки засухи (SPI, индекс Пальмера, ГТК Селянинова) часто недостаточно точны из-за нелинейных взаимосвязей климатических факторов. Предложенная система на основе LSTM-нейросети анализирует многомерные временные ряды и выявляет скрытые закономерности в исторических климатических данных.

**Ключевые преимущества:** точность прогноза 87.4% при горизонте предсказания 90–120 суток, что на 29–35% превосходит традиционные методы; время обработки менее 3 минут на годовой объём данных; интеграция разнородных источников климатических данных в единую нейросетевую архитектуру.

## ✨ Возможности

Система поддерживает загрузку данных из различных форматов (CSV, JSON, XML) с автоматической предобработкой климатических данных, интерполяцией пропусков и расчётом индексов засухи (скользящие средние, дефицит осадков, гидротермический коэффициент). Реализовано обучение LSTM-модели с визуализацией процесса в реальном времени, прогнозирование засушливого года с вероятностной оценкой, детальная оценка качества модели (Precision, Recall, F1-Score, AUC-ROC, матрица ошибок), интерактивная визуализация результатов (графики, диаграммы, тепловые карты), экспорт отчётов в текстовом формате и сохранение обученных моделей.

## 🛠 Технологии

Проект реализован на Python 3.12 с использованием фреймворка TensorFlow 2.16 для глубокого обучения и Keras как высокоуровневого API нейросетей. Графический интерфейс построен на PyQt5. Для обработки и анализа данных применяются NumPy и Pandas, визуализация результатов — Matplotlib, метрики качества и предобработка — Scikit-learn. Дополнительные библиотеки: OpenPyXL для работы с Excel, LXML для парсинга XML, PyInstaller для сборки в исполняемый файл.

## 🚀 Быстрый старт

После запуска приложения перейдите на вкладку «Загрузка и предобработка», нажмите «Загрузить CSV/JSON/XML» и выберите файл с климатическими данными. Установите параметры предобработки: длина последовательности 30 дней (рекомендуется), доля тестовой выборки 0.20 (20%). Нажмите «Выполнить предобработку». Перейдите на вкладку «Обучение модели», настройте параметры: LSTM слои [64, 32], learning rate 0.001, эпохи 100, batch size 32. Нажмите «Начать обучение» и наблюдайте за процессом на графиках в реальном времени. После завершения обучения программа автоматически перейдёт на вкладку «Результаты», где можно просмотреть метрики качества, матрицу ошибок и сохранить отчёт.

## 📁 Структура проекта

Проект имеет модульную структуру: папка `data/` содержит примеры данных (weather_2025_correct.csv), `models/` предназначена для сохранённых моделей (.h5 файлы), `results/` хранит графики и отчёты. Исходный код расположен в `src/`: `data_loader.py` отвечает за загрузку данных из CSV, JSON, XML; `preprocessing.py` выполняет предобработку и нормализацию; `model.py` содержит LSTM модель; `trainer.py` реализует визуализацию обучения; `gui.py` предоставляет графический интерфейс. Точка входа — `main.py`, зависимости зафиксированы в `requirements.txt`, документация — `README.md`.

## 🧠 Архитектура модели

Архитектура LSTM-сети представляет собой последовательную структуру: входной слой принимает тензор размерности (batch_size, time_steps=12, features=24); два скрытых LSTM-слоя с 64 и 32 нейронами соответственно извлекают низкоуровневые и высокоуровневые временные паттерны; слой Dropout (коэффициент 0.3) после каждого LSTM-слоя снижает риск переобучения; полносвязный (Dense) слой с 16 нейронами и функцией активации ReLU выполняет нелинейное преобразование признаков; выходной слой с одним нейроном и сигмоидной функцией активации формирует вероятность принадлежности класса к засушливому году.

Оптимизатор Adam со скоростью обучения 0.001, функция потерь Binary Crossentropy с весами классов для компенсации дисбаланса, метрики оценки: Accuracy, Precision, Recall, AUC. Размер окна последовательности 12 месяцев, количество эпох 100–150 с механизмом ранней остановки (Early Stopping) при отсутствии улучшения валидационной потери в течение 15 эпох, размер мини-выборки 16–32.

## 📊 Результаты

На тестовой выборке модель продемонстрировала следующие результаты: Accuracy = 0.86 (86%), Precision = 0.83 (83%), Recall = 0.81 (81%), F1-score = 0.82 (82%), ROC-AUC = 0.89. Сравнение с традиционными методами показывает преимущество разработанной системы: статистический метод (ГТК Селянинова) имеет точность 52–58% при горизонте прогноза 30–45 суток, спутниковые индексы (NDVI/EVI) — 63–71% при 15–30 сутках, океанические индексы (ENSO/AMO) — 58–65% при 120–180 сутках, тогда как гибридная нейросеть (предлагаемый метод) достигает 87.4% при горизонте 90–120 суток и времени формирования прогноза менее 3 минут.

## 📄 Форматы данных

CSV формат содержит колонки: date (дата), precipitation (осадки в мм), temperature_avg (средняя температура °C), temperature_min (минимальная температура), temperature_max (максимальная температура), humidity_avg (средняя влажность %), pressure_avg (атмосферное давление гПа). JSON формат представляет массив объектов с аналогичными полями. XML формат использует структуру с элементами record, содержащими дочерние элементы для каждого параметра.

## 🔧 Настройка параметров

Параметры предобработки: длина последовательности 30 дней оптимальна для баланса скорости и точности (можно увеличить до 60 дней для долгосрочных трендов), доля тестовой выборки 0.20 является стандартом (можно уменьшить до 0.15 при малом объёме данных или увеличить до 0.30 для более надёжной оценки). Параметры модели: LSTM слои [64, 32] обеспечивают оптимальную ёмкость, learning rate 0.001 — стабильное обучение (0.01 для быстрого обучения с риском нестабильности, 0.0001 для медленного но точного), dropout 0.3 предотвращает переобучение, эпохи 100 с ранней остановкой, batch size 32.

## 🐛 Решение проблем

При ошибке "DLL load failed" переустановите TensorFlow: `pip uninstall tensorflow tensorflow-cpu` затем `pip install tensorflow-cpu==2.16.2`, установите Visual C++ Redistributable. При ошибке "index out of bounds" убедитесь, что в данных присутствуют оба класса (засушливые и не засушливые годы), увеличьте объём данных или измените долю тестовой выборки. При медленной работе используйте `--onedir` вместо `--onefile` при сборке, уменьшите количество эпох или размер последовательности.
